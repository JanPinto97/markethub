Flux de treball Git
main                              → codi estable, només rep merges de dev
└── dev                           → integració de totes les features
    ├── feat/auth-community       → la teva branca base (Fases 2 i 3)
    │   ├── feature/user-model
    │   ├── feature/jwt-auth
    │   └── feature/...
    └── feat/markets              → branca base del teu company (Fase 4)
        ├── feature/ticker
        └── feature/...
Regles: sempre merge --no-ff, esborrar les feature branches un cop mergeades, tag a main quan es completa una fase (v1.0-phase1, etc.), sincronitzar amb dev periòdicament per evitar conflictes grans.

Les 5 fases
Fase 1 — Infraestructura base
Sense dependències, és el punt de partida de tot.

✅ Estructura de carpetes i repositori Git
✅ Backend Express scaffold (Express, Mongoose, dotenv, cors, arquitectura MVC)
✅ Frontend Angular scaffold (standalone components, routing, ApiService)
⬜ Docker setup (posposat per al final del projecte)

Fase 2 — Autenticació
Desbloqueja totes les funcionalitats socials i de la IA.

Model User a MongoDB
Sistema de rols: user (per defecte), moderator i superadmin (creats via script de seed)
Superadmins i moderadors poden eliminar qualsevol post i eliminar usuaris
API: register, login, logout, refresh token, get me
JWT: access token en memòria, refresh token en httpOnly cookie
Límit de 5 intents de login fallits → bloqueig de 15 minuts
Validació de format d'email (sense verificació per correu)
Perfil d'usuari editable: username (únic, obligatori), avatar (URL de text), bio, canvi de contrasenya
Camps públics: username, avatar, bio, data de registre
Camps privats: email, password hash
AuthGuard, AuthInterceptor i renovació automàtica del token al frontend

Fase 3 — Community
La part més complexa en lògica de negoci.

Models: Post, Comment, Community
Posts amb autor, contingut, likes i timestamps
Comunitats públiques i privades (sol·licituds d'entrada)
Admin de comunitat: acceptar sol·licituds, expulsar membres, ancorar posts
Temes de discussió fixes (creats via seed, no moderats)
Feed principal estil Reddit
CRUD complet de posts i comentaris
Navegació lateral amb comunitats pròpies i temes fixes

Fase 4 — Markets (en paral·lel amb la Fase 3)

Integració amb Twelve Data o Alpha Vantage (claus al backend, mai al frontend)
Ticker dinàmic en temps real
Calendari econòmic
Gràfics d'actius amb buscador
Qualsevol nom d'actiu és un enllaç al seu gràfic

Fase 5 — Assistent IA + Home
Al final perquè depèn de tot el contingut anterior.

Widget flotant de chat IA (OpenAI API)
Botons contextuals a tota la web
Pàgina de chat completa estil ChatGPT
Landing page Home amb resums reals de totes les seccions
