Task
Implementar fil de replies recursiu de 3 nivells per PostReddit i PostX. Al nivell 3, mostrar "Continue this thread →" que porta a /community/comment/:commentId. La nova pàgina carrega aquell comentari com a arrel amb el seu subarbre de 3 nivells.

Constraints
Backend:

Substituir la lògica de 2 queries a discussionTopicController.listPostComments i postXController.getComments per una funció buildCommentTree(rootIds, depth=0, maxDepth=3) recursiva que fa Comment.find({ parentComment: { $in: ids } }) per cada nivell
Cada node retornat inclou replies[] i hasMoreReplies: boolean (true si depth >= maxDepth i té fills)
Nova ruta: GET /api/topics/:topicId/posts/:postId/comments/:commentId/thread — retorna el comentari com a arrel amb subarbre de 3 nivells (reutilitza buildCommentTree)

Frontend:

Nou component CommentThreadComponent (standalone): rep @Input() comment, @Input() depth = 0, @Input() maxDepth = 3, @Input() postId; es renderitza recursivament amb <app-comment-thread> per cada reply; si depth >= maxDepth mostra <a [routerLink]>Continue this thread →</a> en lloc de les replies
Indentació: margin-left: depth \* 20px; línia vertical esquerra amb classe depth-line depth-{{depth}}
PostRedditCommentSectionComponent: substituir renderització actual per <app-comment-thread> per cada comentari arrel
PostXCommentSectionComponent (o PostCardComponent): ídem, activar el sistema de replies que ja existia al backend però no es mostrava
Nova pàgina CommentThreadPageComponent a /community/comment/:commentId: crida la nova ruta de thread, mostra breadcrumb "← Back to post", renderitza CommentThreadComponent amb el comentari com a arrel
Afegir ruta a app.routes.ts
CSS custom pur (sense frameworks), consistent amb les variables existents del projecte

Out of scope: no canviar el model Comment.js, no afegir paginació nova, no modificar lògica de likes ni eliminació.

Output format
Retorna només els fitxers modificats o nous, amb el path complet com a títol de cada bloc. No repeteixis fitxers sense canvis.
